# ADR-0001: カタログ駆動のノード組み立て

- 状態: 採用
- 日付: 2026-07-11

## 文脈

部品性能と互換性をNodeの生成処理へ直接埋め込むと、製品追加のたびにコード変更が必要に
なり、購入データと実行時性能がずれる。

## 決定

部品仕様を`parts_catalog.json`と不変Config型で管理し、`NodeBuilder`が部品IDを解決して
互換性を検査し、実行時Deviceへ変換する。

## 結果

製品追加は主にデータ変更で行える。一方、JSON schemaはコード上のloaderと検証処理が
事実上の仕様となるため、項目変更時はloader、設計書、既存データを同時に更新する。
