# Domain documentation

This is a single-context repository.

Before changing domain behavior, read:

- `CONTEXT.md` for the system map and canonical vocabulary.
- Relevant records under `docs/adr/` for prior architectural decisions.

Use glossary terms consistently. Surface conflicts with an ADR instead of
silently overriding it.

ADR content, titles, status, context, decisions, and consequences must be in
Japanese. Keep ADR filenames tool-safe: `NNNN-kebab-case.md` using ASCII.
