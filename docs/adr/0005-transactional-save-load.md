# ADR-0005: 保存データの検証とトランザクショナルな復元

- 状態: 採用
- 日付: 2026-07-13

## 文脈

保存データからNodeを復元すると、Nodeは生成時にworker threadを開始する。読み込み途中で
後続のNodeやNetworkTopologyが不正と判明した場合、Gameへ部分状態を反映すると不完全な
集約が公開され、生成済みNodeを破棄するだけではworker threadも残る。また、保存形式の
versionを検証しなければ、対応外のschemaを現行形式として誤読する可能性がある。

## 決定

読み込みは保存形式versionを最初に検証し、欠落、対応外、破損、不正な項目を公開
`SaveDataError`で拒否する。Inventory、Node、NetworkTopologyは一時状態へ復元し、全項目が
成功した場合だけGameへまとめて反映する。復元途中に失敗した場合は、一時状態で生成した
すべてのNodeを停止してからエラーを返す。

保存時は一時ファイルへの書き込みと置換が成功した場合だけ保存済みversionを進める。
失敗時は既存の保存ファイルを維持し、一時ファイルを削除して元のエラーを返す。

## 結果

呼び出し側は保存データ起因の失敗を一つの例外型で扱え、失敗した読み込みから部分的なGameや
worker threadが残らない。復元処理は一時状態の構築とGameへの反映を分離する必要があり、
新しい保存項目を追加するときはversion検証、失敗時cleanup、round-tripを同時にテストする。
